What does internationalization (i18n) even mean?

  • Translating application messages... that's it right? No!
  • Localizing for each country and region should be easy
  • This is more than just changing locale, currency, date etc.
  • i18n !== translating text from one locale to another
  • Writing app in a way that localizing it for each country and region is easy
  • Integrated process with translators

  • Let's take a look at what the i18n process could look like
  • Developer writes HTML templates (NO ID's!)
  • Templates can be directive or router view tpls in Angular world
  • Messages get extracted in locale they are written in
  • We provide tools for this, but you can integrate them into your toolchain as well
  • Somehow bundled and transformed to file format that translator needs
  • Message bundles are given to translator
  • Translator translate bundles to other locale
  • Translator returns translated bundles in new locale
  • Developer generates new templates from given bundles
  • Again, we provide tools for this
  • As we can see, this process is iterative
  • We might add messages to the template over time etc.

Plurals and Gender

  • We have ngPluralize and ngSwitch, but they're not declarative enough
  • Plural rules vary dramatically across languages.
  • Your translator needs to add or remove categories as they apply to the target locale.
  • They should see all the categories together.
  • We haven't talk about Plurals and Gender yet
  • Plural rules vary: In Russian, it matters whether the least significant digit of the number is 1 (except for 11), [2, 3, 4] (except for 12, 13, 14), or [5, 6, 7, 8, 9, 0].

How can we pluralize declaratively?

{{numMessages, plural,
      =0 { You have no new messages }
      =1 { You have one new message }
   other { You have # new messages }
}}
  • You can write plural and gender based messages (and nest them) in your templates (including attributes.)
  • Advantage: They will be extracted and converted into ICU MessageFormat messages that the translators understand.

What does the translator see?

{numMessages, plural,
      =0 { You have no new messages }
      =1 { You have one new message }
   other { You have # new messages }
}
  • Nearly the same thing but with single curlies and simplified expressions.
  • It's the ICU MessageFormat standard syntax.
  • Translators understand it.
  • Their console automatically provides them the categories to fill in for their target language

Declarative Gender

{{friendGender, select,
       male { Invite him }
     female { Invite her }
      other { Invite them }
}}

Similar to plurals. You can nest them or combine them in other ways.

  • The process of translating is a slow one
  • We need to get feedback faster

Pseudotranslation to the rescue!

Pseudotranslation provides INSTANT translation into gibberish.

  • Sentences size doubled by repeating words
  • Characters munged in some way such as being accented
  • Brackets on either end to easily see text that's cut off

Example

Message:

Enter your name

Pseudotranslated Message:

[Enter Ën̈ẗër̈ your ÿöür̈ name n̈äm̈ë]
  • Catch text that's cut off
  • Catch issues with spacing
  • Catch messages that came from Javascript or were others not extracted
  • Make it part of your workflow (pseudotranslate on save.)
